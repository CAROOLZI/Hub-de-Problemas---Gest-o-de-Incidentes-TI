1. Visão Geral do Sistema
Objetivo: Centralizar a gestão de incidentes técnicos e problemas de infraestrutura em um dashboard dinâmico para facilitar a tomada de decisão.

Público-Alvo: Equipes de suporte técnico e gestores de TI.

2. Requisitos Funcionais (O que o sistema faz)
RF01 - Cadastro de Problemas: O sistema deve permitir o registro de novos incidentes, incluindo título, descrição, categoria (Hardware, Software, Rede) e nível de prioridade.

RF02 - Visualização em Dashboard: O sistema deve exibir gráficos e indicadores de status (Aberto, Em Análise, Concluído) utilizando os dados processados.

RF03 - Filtros de Busca: O usuário deve conseguir filtrar problemas por data, status ou responsável.

3. Requisitos Não Funcionais (Qualidades do sistema)
Desempenho: O dashboard deve carregar os dados em tempo real a partir de consultas SQL otimizadas.

Interface: A interface deve ser intuitiva, seguindo os padrões de usabilidade que você está desenvolvendo no seu capstone.
